# Plan de Migración a Oracle Enterprise en Oracle Linux

Este documento detalla los pasos para migrar la infraestructura de persistencia de PostgreSQL a Oracle Enterprise, manteniendo la Arquitectura Hexagonal del proyecto.

## 1. Preparación del Entorno (Linux VM)
* **OS**: Oracle Linux 8/9.
* **DB**: Oracle Enterprise Edition.
* **Conectividad**: Asegurar que el puerto `1521` esté abierto en el firewall de Linux.
* **Node.js**: El servidor de backend (NestJS) debe tener instaladas las dependencias necesarias.
    * `npm install oracledb`
    * Prisma requiere el driver de Oracle para comunicarse.

## 2. Configuración de Prisma para Oracle
Debido a que Oracle maneja los tipos de datos y las secuencias de forma distinta a PostgreSQL, realizaremos los siguientes cambios:

### Cambio en `schema.prisma`
```prisma
datasource db {
  provider = "oracle"
  url      = env("DATABASE_URL")
}
```

### Notas sobre Tipos de Datos:
* **Autoincrement**: Prisma usará `GENERATED BY DEFAULT AS IDENTITY` en Oracle (disponible desde Oracle 12c).
* **Decimales**: Oracle usa `NUMBER`. Prisma mapeará `Decimal` a `NUMBER`.
* **Strings**: `VarChar(255)` se mantiene igual.

## 3. Estrategia de Carga Masiva (100,000 Registros)
Para cumplir con el requerimiento de carga masiva el viernes, optimizaremos el script `seed-large.ts`.

### Optimizaciones:
1. **Bulk Inserts con Prisma**: Usar `createMany` para todas las entidades, incluyendo Invoices.
2. **Batching**: Agrupar en lotes de 1,000 a 5,000 registros para no saturar el buffer de la base de datos.
3. **Desactivar Constraints Temporalmente (Opcional)**: Si la velocidad es crítica, se pueden desactivar las llaves foráneas durante la carga y reactivarlas después.

## 5. Ejemplo de Conexión (DATABASE_URL)
El formato de la URL para Oracle en Prisma es:
`oracle://user:password@host:1521?service_name=ORCLPDB1`
o usando el SID:
`oracle://user:password@host:1521?sid=ORCL`
